Advanced Dashboard

This dashboard, named "Advanced Dashboard" (Procurement Data), displays a lot of information. It includes navigation panels on the side that can redirect to other dashboards in the workbook, a summary icon on the top left that users can hover over to see the summary of the workbook, parameters in tan that can change the contents of each individual column, filters for the columns, and other specifications that are blurred. On the main page, selecting a cell pops up another table with more in-depth data depending on the selected field.


The actions shown for this workbook display how I can allow fields on the main table to display data from another table based on selection. There are also redirect links for certain fields that redirect the user to a URL in the company's database.

The column selections are built off of 5 parameters with 6 options, one of which is N/A as default to display nothing. The calculated field below displays how the columns will change depending on parameter selection:

CASE [Select Column]
    WHEN "Option 1" THEN [Field 1]
    WHEN "Option 2" THEN [Field 2]
    WHEN "Option 3" THEN [Field 3]
    WHEN "Option 4" THEN [Field 4]
    WHEN "Option 5" THEN [Field 5]
    WHEN "N/A" THEN ""
END

The join for the table was structured similarly to this:

FROM [Database].[Schema].[MainTable] MT  
LEFT JOIN [Database].[Schema].[DetailTable] DT  
    ON MT.[PrimaryKey] = DT.[ForeignKey]  
LEFT JOIN [Database].[Schema].[RelatedTable1] RT1  
    ON RT1.[JoinKey1] = DT.[JoinKey1]  
    AND RT1.[JoinKey2] = DT.[JoinKey2]  
LEFT JOIN [Database].[Schema].[RelatedTable2] RT2  
    ON RT2.[JoinKey3] = RT1.[JoinKey3]  
    AND RT2.[JoinKey4] = RT1.[JoinKey4]  
LEFT JOIN [Database].[Schema].[PeopleTable] PT  
    ON PT.[ID] = RT1.[RequestorID]  
LEFT JOIN [Database].[Schema].[HeaderTable] HT  
    ON RT1.[PONumber] = HT.[PONumber]

These were all left joins because we wanted the first table to be the main source and keep everything joined afterward only relevant to the main table.

There was a calculated field I had to convert into SQL because the calculated field had so many entries to scan that it would lag the interface. So I implemented it into the custom SQL so it would run smoother.

The template for categorizing the content in this one field into groups:

CASE  
    WHEN [Field1] LIKE '%123456%' THEN 'Group - Subcategory'  
    WHEN [Field2] LIKE '%Characters 
    related to item and subcategory%' THEN 'Group - Subcategory'  
    WHEN [Field2] LIKE '%Characters%' AND 
    [Field2] LIKE '%Subcategory%' THEN 'Group - Subcategory'  
    ELSE 'Default Value' -- Optional default case  
END
